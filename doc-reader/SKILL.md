# 文档阅读 Skill

## 概述

帮助用户快速消化文档内容，输出结构化总结 + 重点提炼 + 高质量问题，将文档阅读时间缩短 80%，同时提升理解深度。

---

## 工作流程

### Step 1：获取文档内容

**优先使用浏览器读取（推荐）**：

```
1. 调用 tabs_context_mcp 确认浏览器连接
2. 调用 navigate 导航到文档 URL
3. 调用 get_page_text 提取全文内容
```

如果浏览器未连接：
- 提示用户连接 Claude in Chrome 扩展
- 或请用户粘贴文档内容到对话框

**多文档处理**：逐一读取，分别提取后统一分析。

---

### Step 2：内容分析框架

读取文档后，按以下维度分析：

#### A. 文档类型识别
- **规划类**：战略规划、技术规划、产品规划 → 重点关注目标、路径、优先级
- **分析类**：现状分析、竞品分析、复盘 → 重点关注数据、洞察、结论
- **方案类**：技术方案、产品方案 → 重点关注设计决策、trade-off、风险
- **会议/周报类**：进展更新 → 重点关注进展、阻塞、待决策事项

#### B. 核心信息提取
1. **背景与动机**：为什么写这篇文档？解决什么问题？
2. **核心结论/决策**：最重要的 3-5 个结论是什么？
3. **关键数据与事实**：有哪些具体数字、对比、指标？
4. **行动项与优先级**：P0/P1/P2 是什么？时间节点？
5. **风险与假设**：有哪些未解决的问题或前提假设？

---

### Step 3：输出格式

输出严格遵循以下结构，使用 Markdown 格式：

```markdown
# 📄 [文档标题]

## 一句话概括
> [用一句话说清楚这篇文档在解决什么问题]

## 核心内容总结

### 背景与现状
[2-4句话，说清楚 Why]

### 关键结论 / 决策
[用表格或要点，突出最重要的3-5个结论]

### 规划 / 方案
[按优先级或阶段梳理，保留具体数字和时间节点]

### 重要数据与事实
[数字、对比表格、关键指标]

## 💡 高质量问题

[见 Step 4]
```

---

### Step 4：提出高质量问题

**好问题的标准**：
- ❌ 表面问题："这个功能是做什么的？"
- ✅ 深度问题："这个功能选择 A 方案而非 B 方案，背后的 trade-off 是什么？"

**问题类型矩阵**：

| 类型 | 目的 | 示例 |
|------|------|------|
| **战略对齐** | 挑战假设前提 | "X 和 Y 目标是否存在冲突，如何权衡？" |
| **可行性** | 挑战执行路径 | "里程碑 A 的前提条件是否已具备？" |
| **风险识别** | 挖掘潜在问题 | "如果依赖方 X 不配合，方案如何降级？" |
| **边界条件** | 澄清范围 | "这个方案在什么场景下会失效？" |
| **成功标准** | 明确度量 | "如何量化判断这个目标达成了？" |
| **缺失信息** | 补全盲点 | "文档未提及 X，这是已解决还是待讨论的问题？" |

每次输出 **5-10 个问题**，每个问题需：
1. 说明问题类型
2. 给出具体问题
3. （可选）说明为什么这个问题重要

---

### Step 5：多文档处理

当同时阅读多篇文档时：
1. 分别总结每篇文档
2. 额外增加 **跨文档分析** 部分：
   - 两篇文档的关联与依赖关系
   - 是否存在目标冲突或重叠
   - 协同机会

---

## 注意事项

- **内部文档**需要浏览器登录态，确保 Claude in Chrome 已连接
- **保留原文关键词**：技术术语、产品名称、人名不要翻译或简化
- **数字必须准确**：时间节点、优先级标注（P0/P1/P2）要原样保留
- **避免过度总结**：宁可多保留细节，不要因为"精简"而丢失关键信息
- **问题要有针对性**：问题必须基于文档具体内容，不能是泛泛而问
